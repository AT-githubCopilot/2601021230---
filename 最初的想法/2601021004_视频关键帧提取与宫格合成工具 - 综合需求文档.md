# 一、项目概述

## 1.1 项目目标

开发一款轻量化、易用性强的工具，核心能力为提取MP4视频的关键帧图片，并支持将提取的关键帧合成宫格大图，实现视频核心内容的快速预览与便捷分享，满足普通用户、内容创作者、视频编辑师等群体的日常使用需求。



## 1.2 核心价值

- 高效提取：快速获取视频关键帧，替代手动逐帧截取的繁琐操作；
    
- 便捷分享：通过宫格合成将分散关键帧整合为单张图片，降低分享成本；
    
- 灵活适配：支持多种截取模式与宫格布局，适配不同使用场景与分享平台。
    

## 1.3 目标用户

1. 普通用户：需保存视频精彩瞬间、快速分享视频核心内容；
    
2. 内容创作者：为视频制作预览图、封面图，提升内容传播效率；
    
3. 视频编辑师：快速获取视频素材参考，辅助剪辑工作；
    
4. 研究者/教育工作者：提取教学视频、实验视频的关键步骤，用于汇报或教学。
    

# 二、市场定位与可行性分析

## 2.1 目标用户需求验证

本工具的核心目标用户群体需求真实且刚需，具体验证与场景细化如下：

- 普通用户：核心需求为“快速保存视频精彩瞬间并便捷分享”，典型场景包括旅行vlog关键帧整理、家庭视频精彩片段合集制作（如孩子成长视频宫格图发家庭群），需求特点是操作简单、无需专业知识；
    
- 内容创作者（核心付费/活跃用户）：核心需求为“高效制作视频预览素材提升传播效率”，典型场景包括小红书/朋友圈视频预览图制作、带货视频卖点截图合集、短视频封面备选图生成，需求特点是支持多平台尺寸适配、批量处理；
    
- 视频编辑师：核心需求为“快速获取视频素材参考”，典型场景包括长视频内容脉络梳理、剪辑前关键场景筛选，需求特点是支持无损图片保存、关键帧精准提取；

# 三、核心功能需求

## 2.1 模块一：视频导入与信息解析

### 2.1.1 视频导入功能

- 支持通过“文件选择对话框”选择本地MP4视频文件；
    
- 支持拖拽视频文件至工具窗口直接导入，提升操作便捷性；
    
- 导入后自动校验文件合法性（是否为MP4格式、文件是否损坏、是否可正常解析），非法文件给出明确错误提示（如“文件格式不支持，请选择MP4文件”“文件已损坏，无法解析”）。
    

### 2.1.2 视频信息解析与展示

导入视频后，自动解析并展示以下核心信息，供用户参考：

- 基础信息：文件名、文件大小、文件路径；
    
- 播放信息：播放时长（格式：时:分:秒）、帧率（fps）；
    
- 画面信息：分辨率（宽×高，如1920×1080）、总帧数（计算逻辑：总帧数=播放时长(秒)×帧率）；
    
- 展示形式：以清晰的列表或卡片形式呈现，关键信息（如帧率、总帧数）加粗突出。
    

## 2.2 模块二：关键帧提取

### 2.2.1 提取模式选择

支持4种提取模式，用户可按需选择，默认选中“均匀间隔模式”：

|提取模式|核心逻辑|适用场景|
|---|---|---|
|均匀间隔模式|按“间隔帧数=总帧数÷(截取数量-1)”计算截取点，确保首尾帧必包含，中间帧均匀分布|快速预览视频完整内容脉络|
|关键场景模式|基于画面像素变化程度（差分算法）自动识别关键帧，变化幅度超过阈值则判定为关键帧|提取视频中核心事件、场景切换画面|
|时间点指定模式|用户手动输入多个时间点（格式：时:分:秒），工具精准提取对应时间点的画面|精确提取特定时刻画面（如视频某句台词、某个动作瞬间）|
|混合模式|先按均匀间隔提取基础帧，再补充画面变化明显的关键帧，确保覆盖度与核心场景兼顾|平衡预览完整性与关键场景突出需求|

### 2.2.2 用户参数设置

- 截取数量：用户输入需截取的关键帧数量，强制限制≥2张，输入不满足时给出提示并禁止下一步；
    
- 图片格式：支持JPG、PNG两种格式选择，默认JPG（压缩率高，适合分享），PNG支持无损保存（适合后续编辑）；
    
- 图片质量：仅JPG格式支持，提供3个档位（低/中/高），对应不同压缩比，默认中档；
    
- 保存路径：用户可手动选择文件夹，支持“记住上次保存路径”功能，默认保存至“桌面/视频关键帧”文件夹。
    

### 2.2.3 提取过程与结果展示

- 提取进度：显示实时进度条（百分比+剩余时间预估），提升用户感知；
    
- 中断功能：支持用户点击“取消”按钮中断提取过程，中断后删除已生成的临时文件；
    
- 结果预览：提取完成后，自动打开预览窗口，以缩略图形式展示所有关键帧，支持放大查看单张细节；
    
- 文件命名：默认命名规则为“视频文件名_序号.格式”（如“旅行vlog_01.jpg”），序号按视频时间线顺序排列。
    

## 2.3 模块三：宫格大图合成

作为关键帧提取的可选后置功能，支持用户将提取的全部/部分关键帧合成宫格图；针对关键帧数量较多的场景，可自动拆分为数目更小的若干宫格图合集，避免单张宫格图过于密集，提升视觉观感与预览效率，核心需求为“快速分享”“视觉美观”与“高效预览”。

### 2.3.1 宫格布局设置

- 自动适配布局：根据用户选择的图片数量，自动计算最优矩形宫格（如5张图适配3×2布局、7张图适配4×2布局），不足宫格数量时提供3种填充方式：①空白填充（白色背景）；②重复最后一张图填充；③裁剪首图填充，默认空白填充；
    
- 手动选择布局：提供常见模板（2×2、3×3、4×2、4×3、1×n横向长条、n×1纵向长条），用户可手动选择，选择后实时预览布局效果；
    
- 自定义行列：支持用户手动输入行数和列数（范围：1-10行/列），超出范围给出提示，输入后自动校验“行数×列数≥所选图片数量”，不满足则禁止合成；
    
- 宫格合集拆分配置：针对关键帧数量较多的场景，支持两种拆分模式——①自动拆分：用户可设置单张宫格图的最大图片数量（默认≤9张，即3×3布局，支持2-16张范围内调整），系统自动按时间顺序将所有关键帧分组，生成若干张宫格图组成的合集；②手动分组：用户可在预览窗口手动拖拽关键帧划分分组，自定义每组图片数量与组成，适配个性化预览需求。
    

### 2.3.2 视觉优化配置

- 间距与边框：支持设置图片间距（0-20px，默认5px）、边框粗细（0-3px，默认1px）、边框颜色（提供常见颜色选择+自定义颜色），默认白色间距+浅灰色边框；
    
- 输出尺寸：提供3种预设模板（社交媒体通用：1080×1080px；朋友圈分享：960×960px；长图预览：1920×1080px），支持用户手动输入宽高（单位：px），确保输出图片适配不同分享平台；
    
- 图片适配规则：合成时自动将所有关键帧缩放至统一尺寸，采用“居中裁剪”模式（保证画面主体完整，避免拉伸变形），支持用户切换为“等比例缩放”（保留完整画面，可能产生黑边）。
    

### 2.3.3 个性化定制

- 标题添加：支持在宫格图顶部添加标题（如“视频关键帧预览”“旅行vlog精彩瞬间”），可设置字体、字号、颜色、对齐方式（左/中/右），默认无标题；
    
- 水印添加：支持在宫格图底部添加水印（如工具名称、用户ID、时间戳），可设置水印透明度（0-100%）、字体大小，默认无水印；
    
- 排序调整：预览窗口支持用户拖拽关键帧缩略图，调整宫格内的排列顺序，突破视频时间线限制，优先展示核心画面；
    
- 滤镜效果：提供3种基础滤镜（复古、清新、黑白），支持一键应用于所有宫格图片，使整体风格统一，默认无滤镜。
    

### 2.3.4 合成与保存

- 实时预览：用户调整布局、视觉参数后，实时更新宫格图预览效果，无需反复生成；若开启拆分合集模式，可预览每组宫格图的具体内容与整体排序；
    
- 合成速度：单张宫格图合成时间≤3秒（基于10张1080P图片合成3×4布局）；生成宫格合集时，整体合成时间≤（组数×3秒）；
    
- 保存设置：默认保存路径与关键帧保存路径一致，支持手动修改；默认命名规则为“视频文件名_宫格合集_序号.格式”（如“旅行vlog_宫格合集_01.jpg”），格式与关键帧格式一致；支持勾选“打包保存”，将所有宫格合集图片压缩为ZIP文件，便于整体分享与存储；
    
- 完成提示：合成完成后给出弹窗提示，支持“立即打开文件”“打开保存文件夹”“查看合集整体预览”三种快捷操作。
    

## 2.4 模块四：辅助功能

- 批量处理：支持同时导入多个MP4视频，批量提取关键帧，合成宫格图时可选择“单视频单独合成”或“多视频关键帧合并合成”；
    
- 截取报告：支持生成提取报告（txt格式），包含视频信息、截取参数、关键帧时间点、保存路径等信息，便于后续追溯；
    
- 历史记录：记录最近10次的操作记录（视频路径、截取参数、合成设置），支持一键重新处理；
    
- 帮助指南：内置简易帮助文档，包含核心功能操作步骤、常见问题解答（如“无法解析视频”“合成后图片模糊”）。
    

# 四、非功能需求

## 3.1 兼容性需求

- 系统兼容性：支持Windows 10及以上、macOS 12及以上操作系统；
    
- 视频兼容性：支持主流编码的MP4视频（如H.264、H.265编码），支持分辨率最高4K，最长播放时长120分钟；
    
- 输出兼容性：生成的图片支持主流图片查看工具（如Windows照片、Preview、Photoshop）打开，宫格图支持主流社交媒体平台（微信、微博、小红书）直接上传。
    

## 3.2 性能需求

- 启动速度：工具启动时间≤3秒；
    
- 解析速度：10分钟以内的1080P MP4视频，信息解析时间≤2秒；
    
- 提取速度：1080P视频提取关键帧，单张提取时间≤0.5秒；
    
- 内存占用：处理4K视频时，内存占用≤2GB，避免卡顿或崩溃。
    

## 3.3 易用性需求

- 操作流程：核心流程（导入视频→设置参数→提取→合成）步骤≤5步，无需专业知识即可完成；
    
- 界面设计：简洁直观，核心功能按钮（导入、提取、合成）突出，参数设置区域分类清晰；
    
- 错误提示：所有操作错误（如文件格式错误、参数输入错误、保存失败）均给出明确、易懂的提示，指导用户修正。
    

## 3.4 安全性需求

- 文件安全：处理过程中不修改原始视频文件，提取/合成失败时自动清理临时文件，避免占用存储空间；
    
- 路径安全：禁止保存至系统核心目录（如C盘系统文件夹），避免权限问题导致的保存失败；
    
- 数据隐私：工具仅在本地处理文件，不收集、不上传用户视频、图片等数据，保障用户隐私。
    

# 五、用户操作流程

## 4.1 核心流程（单视频处理）

```Plain
启动工具 → 导入MP4视频 → 查看视频信息 → 选择提取模式 + 设置提取参数（数量、格式等）→ 点击“提取关键帧” → 预览关键帧 → 【可选】选择需合成的关键帧 + 设置宫格参数（含是否拆分合集、单张最大图片数）→ 点击“合成宫格图” → 选择保存方式（单独保存/打包保存）→ 完成（打开文件/文件夹/查看合集预览）
```

## 4.2 批量处理流程

```Plain
启动工具 → 点击“批量处理” → 导入多个MP4视频 → 统一设置提取参数（或单独设置单个视频参数）→ 点击“批量提取” → 预览各视频关键帧 → 【可选】选择合成方式（单视频单独合成/多视频关键帧合并合成）+ 设置宫格参数（含是否拆分合集）→ 点击“批量合成” → 选择保存方式（单独保存/打包保存）→ 完成
```

# 六、数据需求

## 5.1 输入数据

- 视频文件：本地MP4格式文件，支持主流编码；
    
- 用户输入参数：提取数量、图片格式、保存路径、宫格布局、视觉配置等。
    

## 5.2 输出数据

- 关键帧图片：JPG/PNG格式，分辨率与原视频一致；
    
- 宫格图：含单张宫格图与宫格合集（若干张宫格图），均为JPG/PNG格式，分辨率按用户设置；
    
- 提取报告：txt格式，包含视频信息、操作参数等。
    

# 七、优先级说明

## 6.1 高优先级（MVP核心功能）

- 视频导入与信息解析；
    
- 均匀间隔模式关键帧提取；
    
- 基础宫格布局（自动适配+2×2/3×3模板）合成；
    
- 核心参数设置（提取数量、保存路径、图片格式）；
    
- 结果预览与保存。
    

## 6.2 中优先级（体验优化功能）

- 关键场景模式、时间点指定模式提取；
    
- 宫格视觉优化（间距、边框、输出尺寸）及宫格合集拆分功能；
    
- 个性化定制（标题、水印、排序调整）；
    
- 历史记录、帮助指南。
    

## 6.3 低优先级（拓展功能）

- 批量处理功能；
    
- 混合模式提取；
    
- 滤镜效果、截取报告；
    
- 动态宫格（GIF生成）。
